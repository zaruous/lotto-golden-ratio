# 텔레그램봇 배포하기

이 문서는 `telegram-bot/worker.js`의 변경 사항을 Cloudflare Workers에 배포하고 Telegram에서 정상 반영됐는지 확인하는 절차를 설명합니다.

## 1. 사전 준비

- Node.js 18 이상과 npm이 설치되어 있어야 합니다.
- Cloudflare 계정이 있어야 합니다.
- BotFather에서 발급받은 Telegram Bot Token이 필요합니다.
- 명령은 프로젝트 루트에서 실행합니다.

토큰은 소스 코드나 `wrangler.toml`에 작성하지 마세요. Cloudflare Worker Secret으로만 저장합니다.

## 2. 최초 배포

### Cloudflare 로그인

```bash
cd telegram-bot
npx wrangler login
```

브라우저가 열리면 배포할 Cloudflare 계정으로 로그인하고 Wrangler 접근을 승인합니다.

### Telegram 토큰 등록

```bash
npx wrangler secret put TELEGRAM_BOT_TOKEN
```

입력 요청이 나타나면 BotFather에서 발급받은 토큰을 붙여 넣습니다. 이 값은 암호화된 Secret으로 저장되며 `wrangler.toml`에 기록되지 않습니다.

### Worker 배포

```bash
npx wrangler deploy
```

현재 설정의 Worker 이름은 `lotto-smart-bot`입니다. 배포가 완료되면 다음 형식의 주소가 출력됩니다.

```text
https://lotto-smart-bot.<계정-서브도메인>.workers.dev
```

브라우저에서 Worker 주소를 열었을 때 다음 문구가 표시되면 배포가 정상입니다.

```text
Lotto Smart Bot is running!
```

## 3. Telegram 웹훅 최초 연결

Worker를 처음 배포했거나 주소가 변경된 경우 Telegram 웹훅을 등록해야 합니다. `<BOT_TOKEN>`과 `<WORKER_URL>`을 실제 값으로 바꿉니다.

```bash
curl "https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=<WORKER_URL>"
```

예시:

```bash
curl "https://api.telegram.org/bot123456:ABCDEF/setWebhook?url=https://lotto-smart-bot.example.workers.dev"
```

`"ok":true`가 반환되면 연결된 것입니다. 상태는 다음 명령으로 확인합니다.

```bash
curl "https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo"
```

응답의 `url`이 배포한 Worker 주소와 같고 `last_error_message`가 없으면 정상입니다.

> 명령줄에 입력한 토큰은 셸 기록에 남을 수 있습니다. 공용 PC에서는 실행하지 마세요.

## 4. 코드 수정 후 재배포

`telegram-bot/worker.js`를 수정했다면 다음 명령만 다시 실행합니다.

```bash
cd telegram-bot
npx wrangler deploy
```

Worker 이름과 URL이 유지되는 일반적인 재배포에서는 다음 항목을 다시 설정할 필요가 없습니다.

- `TELEGRAM_BOT_TOKEN` Secret
- Telegram 웹훅
- BotFather에서 생성한 봇

배포 명령이 완료된 뒤부터 Telegram 요청은 새 코드로 처리됩니다.

## 5. 명령 테스트

Telegram에서 봇 채팅을 열고 다음 명령을 차례로 실행합니다.

```text
/start
/gen
/gen 10
/gen 50
```

- `/gen`: 기본 5세트를 생성합니다.
- `/gen 10`: 10세트를 생성합니다.
- `/gen 50`: 허용된 최대 개수인 50세트를 생성합니다.
- `/gen 0`, `/gen 51`, `/gen abc`: 사용법 안내를 반환해야 합니다.

50세트처럼 결과가 긴 요청은 Telegram 메시지 길이 제한에 맞춰 여러 메시지로 나뉘어 전송됩니다.

## 6. BotFather 명령 설명 갱신

봇의 메뉴에 표시되는 명령 설명은 Worker 배포만으로 바뀌지 않습니다. 설명을 변경하려면 Telegram에서 `@BotFather`를 열고 다음 절차를 수행합니다.

1. `/setcommands`를 입력합니다.
2. 대상 봇을 선택합니다.
3. 다음 명령 목록을 전송합니다.

```text
start - 봇 소개 및 사용법
gen - 스마트 로또 번호 생성 (기본 5세트, 최대 50세트)
```

## 7. 로그 확인

응답이 없거나 오류가 발생하면 실시간 로그를 실행한 상태에서 Telegram 명령을 다시 보냅니다.

```bash
cd telegram-bot
npx wrangler tail
```

Cloudflare 대시보드의 `Workers & Pages` → `lotto-smart-bot` → `Logs`에서도 실행 기록을 확인할 수 있습니다.

## 8. 자주 사용하는 명령

| 목적 | 명령 |
| --- | --- |
| Cloudflare 로그인 | `npx wrangler login` |
| Bot Token 등록·교체 | `npx wrangler secret put TELEGRAM_BOT_TOKEN` |
| Worker 배포 | `npx wrangler deploy` |
| 실시간 로그 | `npx wrangler tail` |
| 웹훅 확인 | `curl "https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo"` |
| 웹훅 해제 | `curl "https://api.telegram.org/bot<BOT_TOKEN>/deleteWebhook"` |

## 문제 해결

- `401 Unauthorized`: Telegram Bot Token이 잘못됐거나 폐기되었습니다.
- `/gen`이 응답하지 않음: 웹훅 URL과 `getWebhookInfo`의 `last_error_message`를 확인합니다.
- 이전 동작이 계속됨: `npx wrangler deploy`가 성공했는지, 올바른 Cloudflare 계정과 Worker에 배포했는지 확인합니다.
- `TELEGRAM_BOT_TOKEN`이 없음: `npx wrangler secret put TELEGRAM_BOT_TOKEN` 실행 후 다시 배포합니다.
- BotFather 메뉴 설명이 이전 상태임: `/setcommands`로 별도 갱신합니다.
